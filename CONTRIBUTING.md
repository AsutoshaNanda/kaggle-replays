# Contributing

## Welcome

Contributions of all kinds are welcome, whether you are fixing a bug, improving documentation, or proposing a new feature. Anyone is welcome to open an issue or a pull request.

## Reporting Bugs

Please open a GitHub issue using the Bug Report template. Include clear steps to reproduce the problem, the expected behavior, the actual behavior, and screenshots if they help. Do not include any secrets (such as the contents of your `.env` or `auth.json`).

## Suggesting Features

Please open a GitHub issue using the Feature Request template. Describe the problem the feature would solve, your proposed solution, and any alternatives you considered.

## Development Setup

Follow the Getting Started section of [README.md](README.md) to run the project locally. The steps are not duplicated here so they stay in one place.

## Making a Pull Request

1. Fork the repository.
2. Create a branch: `git checkout -b short-description-of-change`.
3. Make your changes.
4. Commit with a clear message describing what changed and why.
5. Push the branch to your fork.
6. Open a pull request against the `main` branch.
7. Fill in the pull request template.
